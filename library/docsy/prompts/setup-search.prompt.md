Help the user add search to a Docsy site. Reference [skill §8](../skills/docsy/SKILL.md#8-search--pick-exactly-one).

## First: pick exactly ONE engine
Never enable two. If the user already has one configured, confirm before swapping and remove the old config.

| Engine | Best when | Builds in `hugo server`? |
|---|---|---|
| **Google Custom Search** (default) | Public site, no infra, fine with Google results page | No (production only) |
| **Algolia DocSearch v3** | Public OSS docs accepted into DocSearch, want instant results | Yes |
| **Lunr (offline)** | Private/intranet site, no external service | Index: production only |

## Google CSE
```yaml
params:
  gcs_engine_id: "<your-engine-id>"
```
Create `content/<lang>/search.md` with `layout: search`. Test with `hugo -e production`.

## Algolia DocSearch v3
```yaml
params:
  search:
    algolia:
      appId: "..."
      apiKey: "..."        # search-only key
      indexName: "..."
```
Customize/disable via `layouts/_partials/algolia/head.html` and `.../scripts.html` (empty file = disabled).

## Lunr (offline)
```yaml
params:
  offlineSearch: true
  offlineSearchSummaryLength: 200
  offlineSearchMaxResults: 25
```
- Per-page opt-out: front matter `exclude_search: true`.
- The index is only generated by `hugo` (production), not `hugo server` — test with `hugo -e production`.

## Custom
Override `layouts/_partials/search-input.html`, `assets/scss/td/_search.scss`, `assets/js/search.js`.

Always verify only one search param family is set, then point the user at [docs/content/search.md](../../docs/content/search.md).
